# BEST_PRACTICES

## Development

- Keep each folder focused on a single purpose.
- Use clear and explicit file names.
- Prefer small, reviewable pull requests.

## API and Data

- Validate incoming request payloads.
- Keep extractor interfaces consistent.
- Use idempotent loads when writing to BigQuery.

## Operations

- Keep deployment steps documented and reproducible.
- Capture incidents and operational learnings in runbooks.
